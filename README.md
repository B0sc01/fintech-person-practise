# Fintech Quant — 量化交易学习系统

一个基于**因子投资**理念的量化分析平台。你可以用它来：浏览和计算股票因子、分析因子有效性、按因子条件筛选股票、对因子策略进行历史回测。

即使你完全不懂量化，跟着下面的教程走一遍，就能理解因子投资的基本逻辑。

## 环境准备

你需要提前安装好以下工具：

| 工具 | 最低版本 | 安装方式 |
|------|---------|---------|
| Python | 3.12+ | [python.org](https://www.python.org/downloads/) 或 `brew install python` |
| Node.js | 18+ | [nodejs.org](https://nodejs.org/) 或 `brew install node` |

安装完成后，在终端输入以下命令验证：

```bash
python --version   # 应显示 Python 3.12.x
node --version     # 应显示 v18.x.x 或更高
```

## 启动系统

在项目根目录下，终端执行：

```bash
./start.sh
```

脚本会自动帮你完成以下事情：
1. 检查 Python 依赖，没有就自动安装
2. 检查 npm 依赖，没有就自动安装
3. 启动后端服务（端口 8000）
4. 启动前端服务（端口 5173）

看到以下输出表示启动成功：

```
=========================================
  系统已启动!

  前端页面: http://localhost:5173
  因子库:   http://localhost:5173/factors
  后端API:  http://localhost:8000/api/v1

  按 Ctrl+C 停止所有服务
=========================================
```

然后在浏览器打开 **http://localhost:5173** 即可看到系统界面。

> 如果你想手动启动（比如调试时），可以分别启动后端和前端，详见[手动启动](#手动启动)。

## 新手教程：从零到第一次回测

下面是完整的使用流程，跟着做一遍你就能理解整个系统。

### 第一步：下载数据

系统需要 A 股的行情数据才能工作。数据来源是 [baostock](http://baostock.com/)，一个免费的证券数据接口。

1. 打开系统后，点击左侧菜单的 **「数据中心」**（Database 图标）
2. 你会看到四个统计卡片：股票数量、日线记录数、数据日期范围、行业数量
3. 点击 **「Download Stock List」** 按钮，等待几秒，下载所有 A 股的基本信息（股票代码、名称等）
4. 下载完成后，股票数量会更新（约 5000+ 只股票）
5. 接着设置日期范围，建议先选最近 **1-3 个月**（数据量小，下载快），点击 **「Download Daily Data」**
6. 下载过程中会显示进度条，等进度到 100% 即可

> **提示**：下载 1 个月的数据大约需要 2-5 分钟，3 个月大约 10-15 分钟。首次使用建议从短时间范围开始。

下载完成后，你会看到：
- 日线记录数变为几十万条
- 数据日期范围显示了实际的起止日期
- 下方的股票列表展示了前 20 只股票

### 第二步：认识因子库

有了数据，我们来看看系统里有哪些"因子"可以使用。

1. 点击左侧菜单的 **「因子库」**（Factor Library）
2. 页面展示了 70+ 个预定义因子，分为 7 大类：

| 类别 | 含义 | 举例 |
|------|------|------|
| **momentum**（动量） | 过去一段时间涨得好的股票未来可能继续涨 | 5日动量、20日动量、60日动量 |
| **value**（价值） | 估值便宜的股票长期收益更高 | 市盈率(PE)、市净率(PB)、盈利收益率 |
| **quality**（质量） | 盈利能力强的公司表现更好 | ROE、ROA、毛利率 |
| **volatility**（波动率） | 波动小的股票风险更低 | 20日波动率、最大回撤、Beta |
| **technical**（技术指标） | 基于价格和成交量的技术分析 | RSI、MACD、布林带 |
| **size**（规模） | 小市值股票可能有超额收益 | 对数市值、换手率 |
| **sentiment**（情绪） | 市场情绪影响短期价格 | 换手率异常、价格缺口 |

3. 你可以在顶部的搜索框输入关键词查找，也可以用下拉框按类别筛选
4. 点击任意一张因子卡片，进入因子详情页

### 第三步：计算因子值

因子只是一套计算规则，需要基于行情数据计算出具体的数值才能使用。

1. 在因子详情页，你会看到因子的完整信息：名称、公式、描述、学术论文引用
2. 页面上方有一个日期范围选择器，默认选了最近 60 天
3. 点击 **「Compute Values」** 按钮
4. 等待几秒到几十秒（取决于数据量和因子复杂度），系统会计算该因子在所有股票上的值
5. 计算完成后，页面会自动刷新，展示：
   - 左侧：因子值的时间序列折线图（取前 50 个数据点）
   - 右侧：最近的因子值表格（股票代码、日期、原始值、Z-Score）

> **什么是 Z-Score？** 计算出的原始因子值经过标准化处理（减去均值，除以标准差），得到 Z-Score。Z-Score 为 0 表示处于平均水平，正值表示高于平均，负值表示低于平均。这使得不同因子之间可以互相比较。

**建议先计算几个常用因子**，后续步骤会用到：
- `momentum_5d`（5日动量）
- `momentum_20d`（20日动量）
- `pe_ttm`（市盈率）
- `volatility_20d`（20日波动率）

### 第四步：分析因子有效性

计算好因子值后，我们来评估这个因子到底有没有用。

1. 点击左侧菜单的 **「因子分析」**（Factor Analysis）
2. 选择一个你已计算过值的因子，设置日期范围
3. 点击 **「Run IC Analysis」**

系统会计算 **IC（Information Coefficient，信息系数）**，它衡量的是：今天的因子值和明天的股票收益之间的相关性。

分析结果包含四个部分：

**IC 序列** — 每天的 IC 值和汇总统计
- **IC Mean**：IC 的平均值，绝对值越大越好（通常 > 0.03 就有一定预测力）
- **IC Std**：IC 的标准差，越小越稳定
- **IR（信息比率）**：IC Mean / IC Std，衡量因子的稳定性，> 0.5 算不错
- **IC 正比率**：IC 为正的天数占比，> 50% 说明因子多数时候有效

**IC 热力图** — 按月展示因子的 IC 表现，颜色越深表示 IC 绝对值越大，一眼看出哪些月份因子表现好。

**相关性矩阵** — 同时选择多个因子，看它们之间的相关性。相关性低的因子组合在一起效果更好（分散风险）。

**分组收益** — 按因子值把股票分成 5 组（分位数），看每组的累计收益。如果第 1 组（因子值最高）的收益明显高于第 5 组（因子值最低），说明这个因子有区分度。

> **怎么看结果？** 如果分位数收益图中，高分位组的曲线一直在上面、低分位组一直在下面，且差距在扩大，说明这个因子是有效的。

### 第五步：用因子筛选股票

确认因子有效后，可以用它来筛选股票。

1. 点击左侧菜单的 **「股票筛选」**（Stock Screener）
2. 页面上默认有一个筛选条件行：
   - 左边选择因子（比如 `momentum_20d`）
   - 中间选择运算符（`>`、`<`、`>=`、`<=`）
   - 右边输入阈值（比如 `0.5`，表示 Z-Score > 0.5）
3. 点击 **「Add Condition」** 可以添加更多条件
4. 顶部选择逻辑关系：
   - **AND**：同时满足所有条件的股票
   - **OR**：满足任意一个条件的股票
5. 点击 **「Search」** 执行筛选

结果表格会展示符合条件的股票，以及每只股票在各个因子上的具体值。

> **举例**：你想找"动量强、估值低"的股票，可以设置两个条件：
> - `momentum_20d > 0.3`（20日动量高于平均）
> - `pe_ttm < -0.5`（市盈率低于平均，因为 PE 的极性是负的）
> - 逻辑选择 AND

### 第六步：策略回测

最后，我们来验证一个因子策略在历史上的表现。

1. 点击左侧菜单的 **「策略回测」**（Backtesting）
2. 配置策略参数：
   - **Factor**：选择一个因子（比如 `momentum_5d`）
   - **Backtest Period**：选择回测时间段（建议至少 1 个月）
   - **Quantiles**：分位数，默认 5（把股票分成 5 组）
   - **Strategy Name**：给策略起个名字
3. 点击 **「Compute」** 先确保因子值已计算（如果第三步已算过可以跳过）
4. 点击 **「Run Backtest」** 启动回测

回测逻辑：每天按因子值把股票分成 N 组，买入因子值最高的一组（top 分位），等权持有，每日调仓。

回测结果包含：

| 指标 | 含义 |
|------|------|
| **Total Return** | 总收益率 |
| **Annual Return** | 年化收益率 |
| **Volatility** | 年化波动率 |
| **Sharpe Ratio** | 夏普比率（每承担一单位风险获得的超额收益，> 1 算好） |
| **Max Drawdown** | 最大回撤（从最高点到最低点的最大跌幅） |
| **Win Rate** | 胜率（收益为正的天数占比） |

页面下方还有一个净值曲线图，横轴是日期，纵轴是累计净值，可以直观看到策略的涨跌过程。

> **提示**：下方有「回测历史」表格，点击任意一行可以查看该次回测的详细结果。

## 各页面详细说明

### 仪表盘（Dashboard）

打开系统的默认首页，包含四张数据卡片和两张图表：

- **Total Stocks**：数据库中股票总数
- **Latest Trade Date**：最新数据日期
- **Avg Close Price**：全部股票最新日的平均收盘价，附带周环比涨跌幅
- **Total Turnover**：最新日的总成交额（单位：亿元）
- **Market Trend 图表**：近 60 个交易日的平均收盘价走势
- **Factor Catalog 图表**：热门因子使用次数排行

### 因子库（Factor Library）

所有可用因子的浏览入口。每个因子卡片展示：名称（中英文）、类别标签、极性（positive = 值越大越好，negative = 值越小越好）、简要描述。

顶部有两个筛选工具：
- **搜索框**：支持按英文名称、代码、中文名称搜索
- **类别下拉框**：按 factor 类别过滤

### 因子详情页

点击因子卡片进入。页面分三个区域：
- **因子信息**：代码、名称、类别、公式（支持数学公式渲染）、描述、学术引用
- **因子值图表**：已计算的因子值时间序列折线图
- **因子值表格**：最近 100 条计算结果（股票代码、日期、原始值、Z-Score）

### 因子分析（Factor Analysis）

因子有效性评估工具。选择因子和日期范围后，可以运行四种分析：
- IC 序列分析（默认）
- IC 热力图
- 相关性矩阵
- 分组收益曲线

### 股票筛选（Stock Screener）

多因子组合选股工具。每行一个条件，支持添加多行。条件之间可以选择 AND（交集）或 OR（并集）。

### 策略回测（Backtesting）

因子策略的历史回测工具。支持配置因子、时间区间、分位数等参数。回测完成后展示统计指标和净值曲线。

### 数据中心（Data Center）

数据管理页面。分左右两区：
- 左侧：数据下载操作（下载股票列表、下载日线数据、进度条）
- 右侧：股票列表预览表格

## 常见问题

### 启动时报错 "address already in use"

端口被占用。执行以下命令后重试：

```bash
lsof -ti:8000 | xargs kill -9
lsof -ti:5173 | xargs kill -9
```

### 页面显示 "无法连接后端"

后端没有启动。确认在项目根目录的 `backend/` 文件夹下运行了启动命令，且终端没有报错。

### 因子计算返回 0 行

可能原因：
1. 没有先下载日线数据 → 去数据中心下载
2. 选择的日期范围不在已下载数据的范围内 → 去数据中心确认数据日期范围
3. 该因子需要的数据字段不完整（比如某些因子需要 PE/PB 数据）

### 筛选结果为空

确认筛选所用的因子已经计算过值（在因子详情页点击 Compute Values）。未计算的因子没有数据，筛选自然没有结果。

### 回测结果显示 "No factor data found"

回测时间范围内没有该因子的计算数据。先去因子详情页计算该因子在对应时间段的值。

### 如何手动启动系统？

**启动后端：**

```bash
cd backend
pip install -r requirements.txt    # 首次需要安装依赖
python -m uvicorn app.main:app --host 0.0.0.0 --port 8000
```

**启动前端（新开一个终端）：**

```bash
cd frontend
npm install                        # 首次需要安装依赖
npx vite --host 0.0.0.0
```

然后访问 http://localhost:5173。
